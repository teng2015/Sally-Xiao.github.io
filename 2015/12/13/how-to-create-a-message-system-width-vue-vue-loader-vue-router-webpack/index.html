如何利用Vue, Vue-loader, Vue-router和webpack构建一个消息系统（一）

发表于 2015-12-13   |   分类于 javascript   |  

背景

Vue是什么?

Vue.js是一个精简的MVVM框架，从技术角度上讲，Vue.js专注于MVVM模型的ViewModel（视图）层。通过双向绑定把View层和Model层连接起来。作者的主要目的是通过一个尽量简单的API产生可反映的数据绑定和可组合的组件。
优点：

  • 1、简单：官方文档很清晰，比Angular简单易学。
  • 2、快速：异步批处理方式更新DOM。
  • 3、组合：用解耦的、可复用的组件组合你的应用程序。
  • 4、紧凑：~18Kb min+gzip,且无依赖。
  • 5、强大：表达式&无需声明依赖的可推导属性(computed properties)。
  • 6、对模块友好：可以通过NPM、Bower或者Duo安装，不强迫所有的代码都遵循Angular约定，使用场景更加灵活。

缺点：

  • 1、项目比较新，没有angular那么成熟。
  • 2、影响度不是很大。
  • 3、不支持IE8，支持度为IE9+。

Vue-loader是什么?

使用Vue.js可以用最简单地方式构建，就是用script标签引入vue.js，然后使用基本的语法来创建函数。但是本文采用的方式是用node构建app的方式，Vue.js给我的优势在于，我可以以一个个组件的方式构建app。每个组件的css代码、javascript代码、模板代码都放在一个文件里。这可以很大程度上提高代码的可维护性。如下图：

这样的文件在我们平时的开发中不常见，构建工具为了可以把这样的文件编译成我们熟悉的css、js、html代码。这就要依赖于vue-loader这个插件。

Vue-router是什么?

vue-router从字面上来讲，他的作用是路由器的功效。什么是路由器呢？容许我来介绍一下：
请看下图：

消息系统一共有关于页面、编辑页面、首页、信息页面、登录页面、注册页面、发送消息页面，当我点击菜单时，系统怎么知道我要跳转到哪个页面呢？我们很容易想到用超链接啊，但是我们这里每个页面其实本质上是一个大的组件，并不存在实体的html,而且我们的消息系统本质上是一个单页应用，不存在多个HTML文件。于是为了让我点击关于按钮时跳转到关于页面，点击首页时跳转到首页，于是我们依赖于vue-router插件，它起一个沟通的作用。

webpack是什么?

webpack是近期最火的一款模块加载器兼打包工具，它能把各种资源，例如JS(含JSX)、coffee、样式（含less/sass）、图片等都作为模块来使用和处理。

我们可以使用require(XXX)的形式引入各模块，即使它们可能需要经过编译(比如JSX和sass), 但我们无须在上面花太多心思，因为webpack有着各种健全的加载器(loader)在默默地处理这些事情。

重点考虑：

  • 1、shimming处理
  • 2、自定义公共模块提取（CommonsChunkPlugin）
  • 3、独立打包样式文件（extract-text-webpack-plugin）
  • 4、使用CDN/远程文件（希望某些文件以\挂载到页面上来加载，但有希望能在webpack的模块中使用上）
  • 5、为了有了webpack还需要gulp打包工具？
